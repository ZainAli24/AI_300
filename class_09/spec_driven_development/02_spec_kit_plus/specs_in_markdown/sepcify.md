# Specify : 
**Specify Phase – Bilkul Simple Definition (Beginner Level)**

---

### **Specify Ka Matlab Sirf Yeh Hai:**

> **"Sirf yeh batao: Kya cheez bana rahe ho aur kyun bana rahe ho."**

- Yeh **technical code nahi**, yeh **user ki zubani baat** hai, yani yha hum technical details (jaise kia tech stack use karen ge) nahe btate, is mien hum ye btate hai ke hum kya bna rahe hai aur kue bna rahe hai yani wo user ki kin problems ko solve kar rha hai.

- Yeh **business level** pe sochna hai, **developer level** pe nahi.

---

### **Focus Hamesha User Par Hota Hai**

| Sawal | Asaan Jawab (Example: Calculator) |
|------|----------------------------------|
| **Ye app kaun use karega?** | Student, teacher, ya ghar mein accounts rakhnay wala insaan. |
| **Ye app unki problem kaise solve karegi?** | Jaldi aur galti se bina hisaab lagaye. |
| **User kaise use karega?** | Buttons dabaye → number type kare → result dikhe. |
| **Success kaise nazar aayega?** | 5 + 3 = 8 dikhe, 10 ÷ 0 = "Error" dikhe, aur sab clear ho. |

---

### **Specify Mein Kya Likhna Hai? (Sirf Yeh 4 Cheezain)**

1. **Kya bana rahe ho?**  
   → *Ek simple calculator jo plus, minus, multiply, divide kare.*

2. **Kyun bana rahe ho?**  
   → *Taake user jaldi hisaab laga sake bina galti ke.*

3. **User kaise use karega? (User Journey)**  
   → *App kholo → numbers type karo → = dabao → result dikhe.*

4. **Success kya hoga? (Acceptance Criteria)**  
   →  
   - 5 + 3 = 8 dikhe  
   - 10 ÷ 0 = "Can't divide by zero" dikhe  
   - Result 6 decimal tak accurate ho

---

### **Specify Mein Kya Nahi Likhna**

- Code kaise likhna hai  
- Python ya Next.js use karo  
- Tests kaise likhna  
- UI kaise design karna  

Yeh sab **baad mein** aayega (Plan, Tasks, Implement mein).

---

### **Command to Run Specify**

```
/sp.specify
Ek interactive calculator banao jo user jaldi hisaab laga sake.
User: buttons dabaye ya keyboard se numbers dale.
Success: result foran dikhe, galti ho to clear message aaye.
```

---

### **Ek Line Mein Samjho**

> **Specify = User ki zubani bolna: "Mujhe yeh chahiye, isliye, aur yeh kaam karega to main khush ho jaunga."**

Bas itna hi!

# **Edge Cases meaning:**
## 🇮🇳 "Edge Case" Ka Matlab: Khaas Aur Ghair Mutawaqqa Soorat-e-Haal

**"Edge case"** ka matlab hai **khaas aur ghair mutawaqqa soorat-e-haal** ya woh soorat-e-haal jo aam halaat se hat kar ho.

Ek misaal ke taur par, ek online shopping app mein **"edge case"** yeh ho sakta hai ki **saarif (user) ek hi waqt mein ek hi cheez ko do mukhtalif alaaton (devices) se kharidne ki koshish kare**. Is soorat mein, app ko yeh faisla karna hoga ki kis **len den (transaction)** ko tarjeeh di jaaye taaki kisi bhi saarif ke saath koi masla na ho.

---

### Tafseel

* **Matlab:** **"Edge case"** aise halaat hain jo **program ya system ke aam istemaal ke cases (use cases) se mukhtalif** hotay hain. Yeh aam taur par **ghair mutawaqqa** ya **bohat kam pesh aanay walay waqiaat** hotay hain jo saarif ke tajarbay (experience) ko mutassir kar saktay hain.

### Misal:

* Agar koi **software developer** ek aisa program bana raha hai jo do numberon ko jama (add) karta hai, toh **"edge case"** yeh ho sakta hai ki saarif **bohat badi tadaad** daakhil (enter) kare, jo program ki salahiyat se **bahar** ho.
* Ek aur misaal hai agar koi **banking app** istemaal karte hue saarif ek hi waqt mein apne phone aur laptop par apna account **log in karne ki koshish** karta hai.

---

### Kyun Zaroori Hai?

* **Behtar Programming:** Yeh zaroori hai ki developer **"edge cases"** par **ghaur** karein taaki woh program ko zyada **qaabil-e-bharosa (reliable)** bana sakein aur ghair mutawaqqa kharabiyon se bach sakain.
* **Behtar Saarif Tajarba:** **"Edge cases"** ko sahih tareeqay se sambhaalne se saarif ka tajarba behtar hota hai kyunki woh kisi bhi soorat-e-haal mein **pur-sakoon aur mehfooz** mehsoos karte hain.

---
